# 기존 서비스 문제 제기

현재 AI 서비스는 많이 사용되고 있지만, 실제 사용 환경에서 불편한 점이 있습니다.

## 1. 개인 파일을 AI와 쓰기 어렵다

- 대부분의 AI는 웹에서 동작하기 때문에 개인 컴퓨터에 있는 파일이나 폴더를 바로 활용하기 힘듭니다.
- 파일을 사용하려면 매번 업로드하거나 복사해야 하고, 벡터 DB나 MCP 같은 기능은 설정이 복잡해 일반 사용자가 접근하기 어렵습니다.

## 2. AI 사용이 장소와 기기에 묶여 있다

- 집 컴퓨터에 파일이 있으면 외부에서는 AI 활용이 어렵고, 휴대폰이나 다른 장소에서 쓰려면 다시 컴퓨터를 켜야 합니다.

## 3. 설정과 환경 구성이 너무 복잡하다

- AI 에이전트를 안전하게 실행하려면 보안, 격리, 권한 설정이 필요하지만 이 과정에는 높은 DevOps 지식이 요구됩니다. 비전문가가 접근하기 어려운 구조입니다.

## 4. 협업 환경과 AI가 분리되어 있다

- 실제 작업과 소통은 Discord 같은 메신저에서 이루어지지만, AI는 별도의 서비스에서 사용됩니다. 이로 인해 작업 요청, 결과 확인, 공유 과정이 끊어집니다.

## 우리의 해결책

이러한 문제를 해결하기 위해 본 프로젝트는 자주 사용하는 **Discord 메신저**에서,
**개인 파일을 안전하게 연결**하고, **복잡한 설정 없이 AI 에이전트를 실행**할 수 있는 환경을 목표로 합니다.
