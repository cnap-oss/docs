# 사용자 가이드

CNAP은 별도의 웹 페이지 없이 Discord 메신저를 기본 사용자 인터페이스(GUI)로 사용하며, 모든 주요 기능은 슬래시 명령어와 모달 입력 화면을 통해 제공됩니다.

## 에이전트 생성

Discord 채널에서 `/agent create` 명령어를 실행하면 에이전트 생성을 위한 모달 입력 창이 표시됩니다.

### 입력 정보

사용자는 다음 정보를 입력합니다:

- **에이전트 이름**: 에이전트를 식별할 수 있는 고유한 이름
- **에이전트 설명**: 에이전트의 목적과 기능에 대한 간단한 설명
- **사용할 AI 모델**: 에이전트가 사용할 AI 모델 선택
- **에이전트 역할 정의(프롬프트)**: 에이전트의 행동과 응답 방식을 정의하는 시스템 프롬프트

::: warning 주의사항
입력 화면에는 민감한 정보 입력을 방지하기 위한 경고 문구가 함께 표시됩니다. API 키나 비밀번호 같은 민감한 정보를 입력하지 마세요.
:::

### 생성 확인

입력을 완료하고 전송하면 새로운 에이전트가 생성됩니다. 생성된 에이전트는 `/agent list` 명령어로 확인할 수 있습니다.

```
/agent list
```

## 에이전트 정보 수정

기존 에이전트의 정보를 수정하려면 `/agent edit` 명령어를 사용합니다.

```
/agent edit
```

### 수정 가능한 항목

- 에이전트 설명
- 사용할 AI 모델
- 에이전트 역할 정의(프롬프트)

::: tip 참고
수정된 내용은 저장 즉시 반영되며 이후 실행부터 적용됩니다. 현재 실행 중인 대화에는 영향을 주지 않습니다.
:::

## 에이전트 실행 및 대화

### 에이전트 호출

`/agent call` 명령어를 실행하면 선택한 에이전트와의 전용 대화 스레드가 자동으로 생성됩니다.

```
/agent call
```

### 대화 스레드

- 각 에이전트 호출마다 새로운 스레드가 생성됩니다.
- 모든 질문과 응답은 해당 스레드 내에서 이루어집니다.
- 에이전트 실행 단위가 분리되어 다른 작업과 섞이지 않습니다.
- 여러 에이전트를 동시에 실행할 수 있으며, 각각 독립적인 스레드에서 동작합니다.

### 대화 방법

스레드 내에서 일반 메시지를 작성하듯이 질문이나 요청을 입력하면 에이전트가 응답합니다.

::: tip 효과적인 대화를 위한 팁

1. **명확한 지시**: 구체적이고 명확한 질문이나 요청을 하세요.
2. **컨텍스트 제공**: 필요한 배경 정보나 파일을 함께 제공하세요.
3. **단계별 진행**: 복잡한 작업은 작은 단계로 나누어 요청하세요.
4. **피드백 제공**: 에이전트의 응답이 부족한 경우 추가 설명을 요청하세요.
   :::

## 에이전트 삭제

더 이상 사용하지 않는 에이전트는 `/agent delete` 명령어로 삭제할 수 있습니다.

```
/agent delete
```

::: danger 주의
에이전트를 삭제하면 해당 에이전트와 관련된 모든 설정이 영구적으로 제거됩니다. 삭제 전 중요한 프롬프트나 설정을 백업하세요.
:::

## 주요 명령어 요약

| 명령어          | 설명                       |
| --------------- | -------------------------- |
| `/agent create` | 새 에이전트 생성           |
| `/agent list`   | 생성된 에이전트 목록 조회  |
| `/agent edit`   | 에이전트 정보 수정         |
| `/agent call`   | 에이전트 실행 및 대화 시작 |
| `/agent delete` | 에이전트 삭제              |

## 문제 해결

### 명령어가 작동하지 않을 때

1. **봇이 온라인 상태인지 확인**하세요.
2. **CNAP 서버가 실행 중인지 확인**하세요.
3. **적절한 권한이 있는지 확인**하세요.
4. Discord를 재시작하거나 몇 분 후 다시 시도하세요.

### 에이전트가 응답하지 않을 때

1. **에이전트 설정을 확인**하세요 (모델, API 키 등).
2. **서버 로그를 확인**하여 오류가 있는지 확인하세요.
3. 새로운 스레드에서 다시 시도하세요.

### 스레드가 생성되지 않을 때

1. **봇이 스레드 생성 권한이 있는지 확인**하세요.
2. **채널 설정에서 스레드 사용이 활성화되어 있는지 확인**하세요.
